# ComfyUI Prompt Extractor

Extracts the **final positive prompt** from PNG images generated by [ComfyUI](https://github.com/comfyanonymous/ComfyUI) — and, as a fallback, from A1111/Forge/SD.Next-style images too.

It analyzes the metadata stored in the PNG and follows the graph into the **last `CLIPTextEncode` node that feeds the `positive` input** of the sampler that created the saved image. Three metadata sources are tried, in order:

1. **`workflow` chunk** — the visual ComfyUI graph (works with `ShowText`, direct `CLIPTextEncode` inputs, and linked string chains).
2. **`prompt` chunk** — the flattened API graph ComfyUI actually executed. This handles cases the visual graph can't, including **subgraphs** (nodes collapsed into subgraphs are already expanded here), reroutes, and images generated via the ComfyUI API.
3. **`parameters` chunk** — A1111/Forge/SD.Next plain-text metadata (positive prompt up to `Negative prompt:` / `Steps:`).

---

## Features

- ✅ Works with **most workflows**, including ComfyUI **subgraphs** and API-generated images.
- ✅ Finds the **exact prompt** that was actually used in sampling.
- ✅ Handles direct text in `CLIPTextEncode` or linked string nodes (`ShowText`, concatenators, etc.).
- ✅ Falls back to A1111-style `parameters` metadata for non-ComfyUI images.
- ✅ Outputs to plain text (`.txt`) or CSV (`.csv`); CSV output is sanitized against spreadsheet formula injection.
- ✅ Optional recursive scan of subfolders (`--recursive`).
- ✅ Deterministic (alphabetical) processing order, per-file skip reasons, and a summary count.
- ✅ Defaults to current folder for input and output.
- ✅ Tested on Windows 11 with Python 3.10+.

## Usage

### Windows quick start

Drag-and-drop a folder onto **`extract_prompts.bat`** — it writes `prompts.txt` and `prompts.csv` into that folder.

### Basic (text output, one prompt per line)

```bash
python extract_prompts.py
```

* Scans the current folder (`.`) for `.png` images.
* Writes `prompts.txt` with one flattened prompt per line.

### CSV output

```bash
python extract_prompts.py --csv
```

* Writes `prompts.csv` with `filename,prompt` columns.
* Prompts are quoted, so commas or newlines are preserved.

### Custom folder

```bash
python extract_prompts.py "C:\path\to\images"
```

### Custom folder + custom output file

```bash
python extract_prompts.py "C:\path\to\images" "C:\output\my_prompts.txt"
python extract_prompts.py "C:\path\to\images" "C:\output\my_prompts.csv" --csv
```

### Recursive scan

```bash
python extract_prompts.py "C:\path\to\images" --recursive
```

* Also scans subfolders; filenames in the output are relative paths.

Run `python extract_prompts.py --help` for the full option list.

## Notes

* If a PNG has no usable prompt metadata, it is skipped and the reason is printed to the console (e.g. `workflow:no-saveimage; prompt:no-prompt-resolved`).
* If multiple `SaveImage` nodes exist, the script uses the one with the **highest execution order** (most likely the one that produced the saved file).
* Prompts in `.txt` output are **flattened to a single line** for easier parsing.
* CSV cells beginning with `=`, `+`, `-`, or `@` are prefixed with `'` so they can't execute as formulas when opened in Excel/LibreOffice.
